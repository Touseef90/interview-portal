C# Interview Question

1. Does C# support multiple inheritance? How you can implement multiple inheritance in C#?
No. However, multiple inheritance can be achieved by using interfaces.
2. What is Abstraction? How to implement abstraction in a real application?
Abstraction means hiding internal implementation details and exposing only the necessary functionality to the user or developer.
For example, when we use the Trim() method in C#, we don't need to know how it actually removes whitespace behind the scenes.
3. What is Polymorphism and what are its types? When to use polymorphism?
Polymorphism allows a single method, class, or interface to behave differently based on the context.
We use it when we want to write generic code that works with different object types. There are 2 types of Polymorphism.
🔹 Compile-time Polymorphism. Method Overloading.
🔹 Run-time Polymorphism. Method Overriding.
4. What is Boxing and Unboxing?
Boxing is the process of converting from value type to reference type.
Unboxing is the process of converting from reference type to value type.
5. When to use Interface and when Abstract class in real application?
🔹 We use an interface when we want to enforce declaration but we usually don't know the exact implementation. And also when we want to support multiple inheritance.
🔹 We use an abstract class when we want to define a base class with common/shared functionality. Or want to provide default implementation for some methods.
6. What is the difference between continue and break statement?
Continue statement, skips current iteration and moves to the next one.
Break statement, end the loop iteration and exit the loop.
7. What is the difference between finally and finalize?
🔹 Finally, it is used in exception handling.
🔹 Finalize is a method which is automatically called by the garbage collector to dispose of objects.
8. What is Garbage Collector (GC)?
The Garbage Collector manages the allocation and release of memory in .NET Framework.

10. How to prevent a class from being inherited?
By using the sealed keyword in class.
11. What is encapsulation?
Encapsulation is the concept of wrapping data and methods into a single unit, and restricting direct access of an object's components.
12. What is the difference between Abstraction and Encapsulation?
🔹 Abstraction means hiding implementation details, showing only essentials.
Goal: Hide complexity
🔹 Encapsulation is hiding data (state) and restricting direct access.
Goal: Protect internal state and ensure safe access
13. What is the difference between Method Overriding and Method Hiding?
🔹 Method Overriding – True Polymorphism
You override a method to customize or extend base class behavior.
🔹 Method Hiding – Replaces Method Using "new"
If you define a method with the same name in the derived class without override, you’re hiding the base method, not overriding it.
14. What is the difference between Abstract class and Interface?
🔹 Abstract - If you need Shared implementation or Fields, constructor, protected members
🔹 Interface - If you need Only behavior definition or Multiple inheritance
15. Can you create an instance of an Abstract class or an Interface?
No. They act as base class via inheritance. Their object creation is not possible.
16. What is the difference between "throw ex" and "throw"?
"throw ex" will change the stack trace, whereas "throw" will preserve the whole stack trace.
17. Explain Generics in C#? When and why to use them in real applications?
Generics in C# allow you to write reusable, and type-safe code. They eliminate the need for casting and reduce duplication by letting you work with any data type using a single class or method.
18. What is the difference between IEnumerable and IQueryable in C#? Why to use IQueryable for SQL queries?
IEnumerable is great for in-memory collections, while IQueryable is designed for remote data sources like databases.
Use IQueryable when working with Entity Framework or LINQ-to-SQL to let the database do the heavy lifting — filtering, joining, sorting — before data comes to your app.
19. What are Extension Methods in C#? When to use extension methods in real applications?
Extension methods allow you to "add" new methods to existing types without modifying the original class or creating a derived class.
They're static methods that enhance readability, reuse, and organization — and are used heavily in LINQ, utility libraries, and clean code patterns.
It looks like you're calling a method on the object itself — but it's really a static method under the hood.
20. What is "this" keyword in C#? When to use it in a real application?
this keyword refers to the current instance of the class — the object whose method or property is being executed.
It is used to resolve naming conflicts, support method chaining, pass the current object, and is required in extension methods.
21. What is the difference between Arraylist and Hashtable?
ArrayList is a dynamic list that stores items by index, while Hashtable stores items by key-value pairs.
Use ArrayList when you just need a resizable list, and Hashtable when you need to quickly look up values by key.

25. What are Generations in garbage collection?
In C#, the garbage collector (GC) uses generations to optimize memory cleanup:
* Generation 0 → Newly allocated short-lived objects.
* Generation 1 → Objects that survived one GC cycle. Acts as a buffer between short-lived and long-lived objects.
* Generation 2 → Long-lived objects that survived multiple collections. Collected least frequently.
💡 Idea: Most objects die young, so GC focuses more on Gen 0 and less on Gen 2 to improve performance.

33. What is AddSingleton, AddScoped and AddTransient method?
🔹 AddSingleton
Lifetime: Only one instance is created and shared throughout the application.
When to use: For stateless services or configurations that are safe to share.
Example:
Logger service
App settings
🔹 AddScoped
Lifetime: One instance is created per HTTP request (shared within the same request, but different for each request).
When to use: For services that store data per request, e.g., repositories.
Example:
Database context (EF Core DbContext)
🔹 AddTransient
Lifetime: A new instance is created every time the service is requested.
When to use: For lightweight, stateless services where you always want a fresh object.
Example:
Helper classes
Data transformers
34. What is the Singleton design pattern?
The Singleton ensures that a class has only one instance throughout the application and provides a global access point to that instance.
35. What is the Factory pattern?
The Factory Pattern centralizes object creation logic. It provides an interface or abstract class for creating objects, but lets the subclasses decide which class to instantiate.
36. How many types of dependency injection?
There are three primary types of dependency injection.
🔹 Constructor Injection
This is the most common and highly recommended type. Dependencies are provided to a class through its constructor.
🔹 Setter Injection (Property Injection)
Dependencies are provided through public setter properties or methods after the object has been instantiated.
🔹 Interface Injection
This is a more complex and rare form where the dependency provides an interface through which it can inject itself into a client. The client implements an interface that contains a method for injecting the dependency.
37. What is the include keyword in LINQ queries?
The Include keyword in LINQ (specifically with Entity Framework) is used for eager loading. It tells the ORM to retrieve related entities along with the main entity in a single database query, preventing the need for additional lazy-loaded queries later.
38. What is Lazy loading, Eager loading?
🔹 Lazy Loading
Related data is loaded automatically and on-demand (i.e., "lazily") when you first access a navigation property.
🔹 Eager Loading
Related data is loaded immediately and upfront (i.e., "eagerly") in a single query along with the main entities.
