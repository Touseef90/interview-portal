React Interview Question

Beginner (Fundamentals)

1. What is React and why is it used? How does it differ from Angular?
React is a JavaScript library for building fast, interactive UIs. Its focus is on the "View" layer. While Angular is a full-fledged framework.
2. What is JSX? Why is it used instead of plain JavaScript?
It allows you to write HTML-like structures in JS. It is more readable.
3. What is the Virtual DOM and how does it work?
A lightweight, in-memory representation of the Real DOM. React uses it to batch updates and minimize expensive direct DOM manipulation. React updates the Virtual DOM first, compares it with a previous snapshot (a process called "diffing"), and then efficiently updates only the necessary parts in the real DOM.
4. What is the difference between functional and class components?
Functional components are simple JavaScript functions that accept props and return JSX. Class components are ES6 classes with more boilerplate, using this.state and lifecycle methods (now mostly legacy).
5. What are Props?
Short for "properties." They are read-only inputs passed from a parent to a child component to make components reusable.
6. What is the State?
An internal object that holds data which may change over time. When the state changes, the component re-renders.
7. Difference between Props and State?
Props are external (passed in) and immutable; State is internal (managed within) and mutable via setState or useState.
8. What is the purpose of keys in React lists?
Unique identifiers that help React track which items in a list have changed, been added, or removed for efficient rendering.
9. What is "Lifting State Up"?
Moving shared state to the closest common parent of components that need to access or modify that data.

Intermediate (Hooks & Component)

10. What are React Hooks, and what are the Rules of Hooks?
Hooks are functions that let you "hook into" React state and lifecycle features from functional components. It must be called at the top level (not inside loops/conditions) and only from React functional components or custom hooks.
11. What is the useState Hook?
useState is a Hook that lets you add React state to functional components. It returns an array with two elements: the current state value and a function to update it.

13. What is the useEffect Hook and how does it work?
useEffect lets you perform side effects in functional components, such as data fetching, setting up a subscription, or manually changing the DOM. It replaces lifecycle methods like componentDidMount, componentDidUpdate, and componentWillUnmount.
Used for side effects (API calls, subscriptions, DOM updates). Mention the dependency array: [] (mount only), [val] (on change), or no array (every render).
14. What is the difference between useEffect and useLayoutEffect?
useLayoutEffect and useEffect. useLayoutEffect runs synchronously after DOM mutations but before the browser paints; useEffect runs asynchronously after the paint.
useLayoutEffect fires synchronously after all DOM mutations but before the browser has a chance to paint. It's useful for reading layout from the DOM and synchronously re-rendering, but it blocks visual updates. useEffect runs after paint and is preferred for most side effects to avoid blocking.
15. How does React handle re-rendering?
React handles re-rendering through a three-step cycle designed to keep the user interface in sync with the application state: Trigger, Render, and Commit.
1. The Trigger Phase
A re-render is always initiated by a change in data.
2. The Render Phase
In this phase, React calls your component functions to determine what the UI should look like.
3. The Commit Phase
After calculating the differences, React applies them to the real DOM.
16. What is useMemo vs. useCallback?
useMemo memoizes a value (result of a calculation); useCallback memoizes a function definition to prevent unnecessary child re-renders.
useMemo returns a memoized value. It only recomputes the value when its dependencies change, helping to optimize expensive calculations and avoid them on every render.
useCallback returns a memoized version of a callback function that only changes if one of its dependencies has changed. This is useful to prevent unnecessary re-renders of child components that rely on reference equality to skip updates.
17. How do you handle forms in React? What are controlled vs uncontrolled components?
Handling forms in React involves managing input data and submission events. There are two primary architectural patterns: Controlled and Uncontrolled components.
In a controlled component, form data is handled by React state. In an uncontrolled component, form data is handled by the DOM itself, and you use a ref to access the values.
18. What is conditional rendering?
Conditional rendering works the same way conditions work in JavaScript. You can use if statements, the ternary operator (? :), or the logical AND operator (&&) to render different UI elements based on specific conditions.
19. What are fragments in React?
A wrapper that lets you group multiple elements without adding an extra node to the HTML DOM. This is useful to avoid unnecessary wrapper like "div".
20. What is Context API and when would you use it?
A way to share global data (like themes or auth) without "prop drilling" through every level of the tree.
The Context API provides a way to share values like the authenticated user, theme, or preferred language across the entire component tree without having to pass props down manually at every level.
21. What is useRef?
useRef returns a mutable ref object whose .current property persists for the full lifetime of the component without causing re-renders.
useRef returns a mutable ref object whose .current property is initialized to the passed argument. It's commonly used to access DOM elements directly or to persist values across renders without causing a re-renders.
22. What is the useReducer Hook, and how is it different from useState?
An alternative to useState for complex state logic, often used when the next state depends on the previous one (similar to Redux logic).
23. What is the "Prop Drilling" problem?
Props drilling occurs when you need to pass data through multiple intermediate components just to get it to a deeply nested child component. You can avoid it by using the Context API or a state management library like Redux.
24. How does the React.memo function work?
A higher-order component that prevents a functional component from re-rendering if its props haven't changed.

Advanced (Architecture & Performance)

25. What are Higher-Order Components (HOC)?
A Higher-Order Component is a pattern where a function takes a component and returns a new component with additional props or functionality. It's an advanced technique for reusing component logic.
26. What is Redux and how is it different from Context API for state management?
While Context API is great for prop drilling and simple global state, Redux is better for complex applications requiring a single source of truth, predictable state updates with reducers, powerful middleware for async logic, and time-travel debugging.
27. What is reconciliation in React?
The algorithm React uses to "diff" the Virtual DOM trees and determine which parts of the Real DOM need updating.
Reconciliation is the process React uses to update the DOM. When a component's props or state change, React builds a new virtual DOM tree and compares it to the previous one (a process called "diffing"). It then calculates the most efficient way to update the real DOM with these changes.
28. What are custom hooks?
Functions starting with "use" that allow you to extract and reuse component logic (e.g., useFetch, useLocalStorage).
29. What is lazy loading, Suspense and code splitting in React?
Breaking a large bundle into smaller chunks using React.lazy and Suspense so users only load the code they need.
30. How does React Router work?
React Router is a library that enables client-side routing in React applications, allowing them to function as Single-Page Applications (SPAs). It simulates a multi-page experience by updating the URL and UI dynamically without a full page reload from the server.
31. What are Error Boundaries?
Error Boundaries special class components that catch JS errors in their child tree and display a fallback UI.
32. What is Concurrent Mode?
A set of features (like useTransition and useDeferredValue) that allow React to stay responsive by interrupting long-running renders.
33. What are React Server Components (RSC)?
Components that execute on the server, sending zero JavaScript to the client, which significantly improves load times.
34. What is Hydration?
The process where React attaches event listeners to static HTML sent by the server (SSR), making the page interactive.
35. Difference between SSR and CSR?
Server-Side Rendering (SSR) generates HTML on the server (better SEO/First Paint); Client-Side Rendering (CSR) builds the UI in the browser.
36. How would you optimize a list with 10,000 items?
Use Windowing or Virtualization (e.g., react-window) to only render the items currently visible in the viewport.
37. How would you design a custom useDebounce Hook?
A useDebounce Hook would delay updating a value until after a specified delay, reducing the number of renders from rapidly firing events like keystrokes.
38. Explain useTransition and useDeferredValue. How do they improve user experience?
These are React 18 Hooks for concurrent rendering.
useTransition allows you to mark a state update as non-urgent, keeping the UI responsive during heavy renders.
useDeferredValue lets you defer updating a less important part of the UI, similar to debouncing but built-in.
39. Explain how React's automatic batching works?
Batching is when React groups multiple state updates into a single re-render for better performance. In React 18, automatic batching is enabled for all updates, including those inside promises, timeouts, and event handlers, ensuring efficient rendering.
40. What are the performance implications of using an anonymous function as a prop ( e.g., onClick={() => doSomething()} )?
This creates a new function on every render. If this prop is passed to a memoized child component (React.memo), the child will see a new function reference each time and re-render unnecessarily, defeating the purpose of memoization. Using useCallback solves this by maintaining a stable reference.
