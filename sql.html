SQL Interview Question

1. What is the difference between DBMS and RDBMS?
DBMS:
  • It stores data as file.
  • No relationshipt between data.
RDBMS:
  • It stores data in TABULAR form.
  • Data stored in tables are RELATED to each other.
2. What is a Constraint in SQL? What are its type?
SQL constraints are used to specify rules for the data in a table. Constraints are used to limit the type of data that can go into a table. For example: PRIMARY KEY, NOT NULL, FOREIGN KEY, CHECK, DEFAULT, UNIQUE etc.
3. What is a difference between Primary key and Unique key?
Primary Key:
  • Primary Key can't accept NULL value.
  • Creates clustered index.
  • Only one Primary key in a Table.
Unique Key:
  • Unique Key can accept only one NULL value.
  • Creates non-clustered index.
  • More than one Unique key in a Table.
4. What are Triggers and types of triggers?
Triggers are stored programs, which are AUTOMATICALLY executed or fired when some events (INSERT, UPDATE, or DELETE) occur.
There 2 types of triggers in DML (Instead of triggers, and After triggers).
AFTER Triggers: These fire after the DML operation (INSERT, UPDATE, or DELETE).
INSTEAD OF Triggers: Instead of the triggering DML operation they skip (INSERT, UPDATE, or DELETE) and execute other statement defined in the trigger.
5. What is a View?
A view is a VIRTUAL table which consits of a subset of data contained in a table or more than one table.
6. What is the difference between HAVING clause and WHERE clause?
1. WHERE is used before GROUP BY clause and HAVING is used after GROUP BY clause.
2. WHERE cannot contain aggregate function but HAVING can contain.
7. What is sub query or nested query or inner query in SQL?
It is a query within another SQL query and embedded within the WHERE clause.
8. What is auto increment / identity column in SQL Server?
Auto increment allows a unique number to be generated automatically when a new record is inserted into a table. Mostly it is the primary key only.
9. What are joins in SQL?
A JOIN clause is used to combine rows from two or more tables, based on related column between them.
10. What are the types of joins in SQL Server?
  1. INNER JOIN - Only rows with matching values in both tables.
  2. LEFT JOIN - All rows from the left table + matching rows from the right table.
  3. RIGHT JOIN - All rows from the right table + matching rows from the left table.
  4. FULL OUTER JOIN - All rows from both tables, pairing them where matches exist.
11. What is a self-join?
A self-join is a join in which a table is joined to itself. It is useful when the data in a table has a relationship with other rows in the same table.
Example: Suppose there is an 'Employees' table with the columns 'EmployeeID', 'EmployeeName', and 'ManagerID'. The 'ManagerID' column stores the 'EmployeeID' of the employee's manager. If we want to retrieve each employee's name along with their manager's name, we can join the 'Employees' table to itself. This type of join is called a 'self-join'.
12. What are indexes in SQL Server?
SQL indexes are used in relational databases to retrieve data very fast.
13. What is clustered index?
A clustered index determines the physical order of data rows in a table and stores the actual data directly within the index itself. Because data can only be physically sorted in one way, a table can have only one clustered index. It is like a dictionary.
14. What is non-clustered index?
A non-clustered index is stored at one place and table data is stored in another place. So the index is not physically stored. It contains the index key values and each key value entry has a pointer to the data row that contains the key value. A table can have multiple non-clustered index. It is like a book index at the start of the page 'Table of Contents'.
15. What is the difference between clustered and non-clustered index?
  1. A clustered index determines the physical order of data rows and non-clustered index is stored at one place and table data is stored in another place.
  2. A table can have only one clustered index but it can have multiple non-clustered index.
16. How to create clustered and non-clustered index in a table?
When you create a PRIMARY KEY constraint, a clustered index on the column or columns is automatically created. For creating clustered and non-clustered index explicitly below is the query.
CREATE CLUSTERED INDEX "index_name" ON "table_name" ( "column_name" ASC/DESC ) CREATE NONCLUSTERED INDEX "index_name" ON "table_name" ( "column_name" ASC/DESC )

The column after WHERE condition, which is 'name' here.
18. What is the difference between Stored Procedure and Function?
  1. SP may or may not return a value but function must return a value.
  2. SP can have input/output parameters but function only has input parameters.
  3. We can call function inside SP but cannot call SP inside a function.
  4. We cannot use SP in SQL statements like SELECT, INSERT, UPDATE, DELETE, etc. But we can use them with function. SELECT *, dbo.fnCountry(city.long) FROM city.
  5. We can use try catch exception handling in SP but we cannot do that in function.
  6. We can use transactions in SP but we cannot do that in function.
19. How to optimize a Stored Procedure or SQL Query?
  1. Use SET NOCOUNT ON
  2. Specify column name instead of using * in SELECT statement.
  3. Use schema name before objects or table names. Example: SELECT id, name FROM dbo.Employee
  4. Do not use DYNAMIC QUERIES. They are vulnerable to SQL Injections.
  5. Use EXISTS() instead of COUNT().
  6. Use TRANSACTIONS only when required.
20. What is a Cursor? Why to avoid them?
A database Cursor is a control which enables traversal/iteration over the rows or records in the table. It occupies lot of memory and not good for performance.
21. What is the difference between scope_identity and @@identity?
  1. Both are used to get the last value entered in the identity column of the table.
  2. The @@identity returns the last identity created in the same session. The sessoin is DB connection.
  3. The scope_identity() function return the last identity created in the same session and the same scope. The scope is the current query or the current SP.
  4. Normally we have to use scope_identity() function inside SP.
22. What is CTE in SQL Server?
A CTE (Common Table Expression), is a temporary named result set, that you can reference within a SELECT, INSERT, UPDATE, or DELETE statement. Think of it as a virtual table that is created on the fly.
23. What is the difference between DELETE, TRUNCATE, and DROP commands?
The primary difference is that DELETE removes specific or all rows from a table using a filter, TRUNCATE empties a table completely by deallocating its data pages, and DROP completely deletes both the table structure and its data from the database.
24. How to get the Nth highest salary of an employee?
  1. The logic is first SELECT TOP N salaries in descending order.
    SELECT DISTINCT TOP N salary FROM dbo.Employees ORDER BY salary DESC
  2. Put the result in 'RESULT' then order by ASC
    SELECT salary FROM ( SELECT DISTINCT TOP N salary FROM dbo.Employees ORDER BY salary DESC ) RESULT ORDER BY salary
25. What are ACID properties?
ACID property are used when you are handling transactions in SQL. That guarantee reliable transaction processing and maintain database data integrity.
26. What are Magic Tables in SQL Server?
They are temporary, memory-resident virtual tables automatically created and managed by the database engine to hold data modified during Data Manipulation Language (DML) operations.
