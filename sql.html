SQL Interview Question

1. What is the difference between DBMS and RDBMS?
DBMS:
  • It stores data as file.
  • No relationshipt between data.
RDBMS:
  • It stores data in TABULAR form.
  • Data stored in tables are RELATED to each other.
2. What is a Constraint in SQL? What are its type?
SQL constraints are used to specify rules for the data in a table. Constraints are used to limit the type of data that can go into a table. For example: PRIMARY KEY, NOT NULL, FOREIGN KEY, CHECK, DEFAULT, UNIQUE etc.
3. What is a difference between Primary key and Unique key?
Primary Key:
  • Primary Key can't accept NULL value.
  • Creates clustered index.
  • Only one Primary key in a Table.
Unique Key:
  • Unique Key can accept only one NULL value.
  • Creates non-clustered index.
  • More than one Unique key in a Table.
4. What are Triggers and types of triggers?
Triggers are stored programs, which are AUTOMATICALLY executed or fired when some events (INSERT, UPDATE, or DELETE) occur.
There 2 types of triggers in DML (Instead of triggers, and After triggers).
AFTER Triggers: These fire after the DML operation (INSERT, UPDATE, or DELETE).
INSTEAD OF Triggers: Instead of the triggering DML operation they skip (INSERT, UPDATE, or DELETE) and execute other statement defined in the trigger.
5. What is a View?
A view is a VIRTUAL table which consits of a subset of data contained in a table or more than one table.
6. What is the difference between HAVING clause and WHERE clause?
1. WHERE is used before GROUP BY clause and HAVING is used after GROUP BY clause.
2. WHERE cannot contain aggregate function but HAVING can contain.
7. What is sub query or nested query or inner query in SQL?
It is a query withing another SQL query and embedded within the WHERE clause.
8. What is auto increment / identity column in SQL Server?
Auto increment allows a unique number to be generated automatically when a new record is inserted into a table. Mostly it is the primary key only.
9. What are joins in SQL?
A JOIN clause is used to combine rows from two or more tables, based on related column between them.
10. What are the types of joins in SQL Server?
  1. INNER JOIN - Only rows with matching values in both tables.
  2. LEFT JOIN - All rows from the left table + matching rows from the right table.
  3. RIGHT JOIN - All rows from the right table + matching rows from the left table.
  4. FULL OUTER JOIN - All rows from both tables, pairing them where matches exist.
